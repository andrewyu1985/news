# Настройка Telegram: бот и публикация в канал

Документация по настройке Telegram-бота и публикации дайджестов в канал для News Digest Pipeline.

---

## 1. Telegram Bot

Бот создан через [@BotFather](https://t.me/BotFather): **@aivideoproducer_bot**

Токен бота хранится в переменной окружения `TELEGRAM_BOT_TOKEN`.

Бот выполняет две функции:
- **Приём URL от пользователя** — через webhook принимает ссылки на статьи и сохраняет их в базу данных
- **Публикация дайджестов в канал** — отправляет готовые дайджесты в Telegram-канал

---

## 2. Webhook для приёма URL

### Адрес webhook

```
https://news.questtales.com/api/telegram/webhook
```

Webhook регистрируется автоматически при запуске сервера через Telegram Bot API (`setWebhook`). Для этого необходимо задать переменную `BASE_URL`.

### Защита

Webhook защищён секретным токеном. Telegram передаёт его в заголовке `X-Telegram-Bot-Api-Secret-Token`. Сервер проверяет значение заголовка против переменной `TELEGRAM_WEBHOOK_SECRET`. При несовпадении запрос отклоняется с кодом 403.

### Безопасность: фильтрация по chat ID

Бот принимает сообщения **только** от пользователя с указанным `TELEGRAM_CHAT_ID` (личный чат: `196267257`). Сообщения от других пользователей или чатов отклоняются с записью в лог.

### Обработка сообщений

Бот извлекает URL из текста сообщений с помощью регулярного выражения и сохраняет каждый уникальный URL в базу данных как статью с источником `telegram`. Дубликаты обнаруживаются и не сохраняются повторно.

После сохранения бот отвечает сообщением с количеством сохранённых/дубликатов и текущим числом новых статей.

### Уведомление о достижении порога

Когда количество новых статей достигает порога (по умолчанию 13, настраивается через `ARTICLE_THRESHOLD`), бот отправляет уведомление о том, что накопилось достаточно статей для генерации дайджеста.

### Команды бота

| Команда | Описание |
|---------|----------|
| `/start`, `/help` | Справка по боту |
| `/status` | Количество статей по статусам (новые, в обработке, использованные, всего) |
| `/generate` | Принудительная генерация дайджеста из накопленных статей |

## 3. Публикация в канал

### Канал

- Название: **@alexkrol** (Квест Теория Каст и Ролей)
- Channel ID: `-1001094435752`

**Важно:** ID канала должен включать префикс `-100` для supergroup/channel. Без этого префикса Telegram API вернёт ошибку "chat not found".

### Настройка прав

Бот должен быть добавлен как **администратор** канала с разрешением **Post Messages** (Публикация сообщений). Без этого разрешения бот не сможет отправлять сообщения в канал.

### Переменная окружения

Для публикации используется отдельная переменная `TELEGRAM_PUBLISH_CHAT_ID`, которая указывает на канал. Это **не то же самое**, что `TELEGRAM_CHAT_ID` (личный чат для приёма URL).

## 4. Разбивка длинных сообщений

Telegram ограничивает длину одного сообщения **4096 символами**. Дайджесты обычно занимают 9000-15000 символов, поэтому их необходимо разбивать.

### Алгоритм разбивки

1. Если текст <= 4096 символов — отправляется одним сообщением
2. Иначе ищется последняя граница пронумерованного пункта в пределах лимита (паттерн `\n\n5. `, `\n\n12. ` и т.д.)
3. Если граница пункта не найдена — разрез по последнему двойному переносу строки (`\n\n`)
4. Крайний случай — жёсткий разрез на позиции 4096

### Отправка частей

- Части отправляются последовательно с задержкой **1 секунда** между сообщениями
- Параметр `disable_web_page_preview: true` — превью ссылок отключено, чтобы не загромождать ленту канала
- Функция возвращает ID первого сообщения и общее количество отправленных частей

## 5. iOS Shortcut для отправки URL

Для быстрой отправки ссылок из iPhone используется iOS Shortcut, интегрированный в Share Sheet.

### Как работает

1. Пользователь открывает статью в браузере или приложении
2. Нажимает "Поделиться" (Share Sheet)
3. Выбирает Shortcut
4. Shortcut отправляет URL боту в Telegram

### Настройка Shortcut

- Источник переменной: **Ввод из быстрой команды** (Shortcut Input)
- Shortcut получает URL из Share Sheet и отправляет его в чат с ботом

### Особенность iOS 26

В iOS 26 опция "Show in Share Sheet" переименована в **"На странице экспорта"**. При настройке Shortcut ищите эту опцию в свойствах быстрой команды.

## 6. Переменные окружения

| Переменная | Назначение | Пример |
|------------|-----------|--------|
| `TELEGRAM_BOT_TOKEN` | Токен бота от @BotFather | `123456:ABC-DEF1234ghIkl-zyx57W2v1u123ew11` |
| `TELEGRAM_CHAT_ID` | ID личного чата пользователя (для приёма URL) | `196267257` |
| `TELEGRAM_PUBLISH_CHAT_ID` | ID канала (для публикации дайджестов) | `-1001094435752` |
| `TELEGRAM_WEBHOOK_SECRET` | Секретный токен для верификации webhook | произвольная строка |

Все переменные задаются в `.env` файле или в `docker-compose.yml`.

## 7. Troubleshooting

### "chat not found" при публикации в канал

**Причина:** ID канала указан без префикса `-100`.

**Решение:** Для supergroup и каналов Telegram требует префикс `-100` перед числовым ID. Например, если ID канала `1094435752`, то в переменной должно быть `-1001094435752`.

### Бот не может отправить сообщение в канал

**Причина:** Бот не добавлен как администратор канала или не имеет разрешения на публикацию.

**Решение:**
1. Откройте настройки канала в Telegram
2. Перейдите в "Администраторы"
3. Добавьте бота @aivideoproducer_bot
4. Включите разрешение "Post Messages"

### Сообщения от других пользователей отклоняются

**Причина:** `TELEGRAM_CHAT_ID` не совпадает с ID чата отправителя.

**Решение:** Убедитесь, что `TELEGRAM_CHAT_ID` содержит ваш **личный** chat ID (`196267257`), а не ID канала. Узнать свой chat ID можно, отправив сообщение боту [@userinfobot](https://t.me/userinfobot).

### Webhook не регистрируется

**Причина:** Не задана переменная `BASE_URL` или `TELEGRAM_BOT_TOKEN`.

**Решение:** Проверьте, что обе переменные заданы. Webhook регистрируется автоматически при старте сервера. В логах должна появиться строка `[telegram-bot] Webhook set: <url>`.

### Бот не отвечает на сообщения

**Возможные причины:**
- Webhook не зарегистрирован (проверьте логи сервера)
- Сервер недоступен по HTTPS (Telegram требует валидный SSL-сертификат)
- Неверный `TELEGRAM_WEBHOOK_SECRET` (проверьте заголовок `X-Telegram-Bot-Api-Secret-Token`)

### Проверка состояния webhook

Для диагностики можно запросить текущий статус webhook через Telegram API:

```bash
curl https://api.telegram.org/bot<BOT_TOKEN>/getWebhookInfo
```

Ответ покажет текущий URL webhook, количество ошибок и последнюю ошибку.
